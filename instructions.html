Generate your key pair

Generate your key pair.

  • Open a terminal/command line interface
  • Navigate to the folder containing the downloaded client app
  • Execute the following command:
     > oal gen 
  • This will generate your personal private/public key pair and download the master public key. You will receive the following message on success:
     xxx bytes written successfully to ./keys/public.key        
     xxx bytes written successfully to ./keys/private.key       
     xxx bytes written successfully to ./keys/master-public.key 

Register your identity

After you have generated your keys, you need to register your identity. Each identity is unique and must adhere to the following rules:

  • Must start with a lowercase letter
  • Minimum length 6 character
  • maximum length 50 characters
  • Limited to lowercase letters, numbers, underscore, and dash characters
Register your identity.
  • Open a terminal/command line interface
  • Navigate to the folder containing the downloaded client app
  • Execute the following command:
     > oal reg <username> 
  • This will generate your identity (the only possible PII, depending on your identity name). You will receive the following message:
     Result: 200, Reason: Created 
     API Version: (prod)v.x.x.x   
  • If you have chosen an identity name that already exists you will receive the following message. Choose another identity name.
     Result: 409, Reason: Already Exists 
  • If you attempt to register a second identity, you will receive the following message:
     Result: 402, Reason: Payment Required. Only one free registration per user. 
  • Users only can have one free identity. You can choose to have a credit card associated with your account for purchase of more identities and other services. See instructions below on linking a credit card.

Follow other identities

In order to send files to another user you must follow their identity. This, obviously, requires you to know the name of their identity. You can follow any number of identities for free.
Follow other identities.

  • Open a terminal/command line interface
  • Navigate to the folder containing the downloaded client app
  • Execute the following command to follow an identity:
     > oal follow <identity> 
  • If the identity exists, you will receive the following message. You will receive the identity's public key for encryption.
     xxx bytes written to ./following/<identity>/public.key 
     Result: 200, Reason: OK                                
     API Version: (prod)v.x.x.x                             
  • If the identity does not exist, you will receive the following message:
     Result: 404, Reason: Not Found 
     API Version: (prod)v.x.x.x     

Send Files

After you follow an identity, you can send files to it. These files are encrypted using the identity's public key that you obtain when you follow it.
Send Files.

  • Open a terminal/command line interface
  • Navigate to the folder containing the downloaded client app
  • Execute the following command to send a file to an identity:
     > oal send <identity> <path to file> -f 
  • If the transfer is successful, you will receive the following message.
     Result: 201, Reason: Sent  
     API Version: (prod)v.x.x.x 
  • If you are not following the identity, you will receive the following message:
     Result: 400, Reason: You Are Not Following <identity> 
  • You can also send plain text not in a file to an identity. Execute the following command. Be sure to include the quotes. All text between the quotes will be sent, not including the quotes. If successful, you will receive the same message as above.
     > oal send <identity> "message to send" 
  • If you attempt to send more than the allotted free transactions, you will receive the following message. The transaction can be accomplished if you authorize a credit card payment (see below).
     Next call available at: Month day, year at hour:minute (UTC)                 
     Result: 402, Reason: Payment Required $x.xx USD. One free call per xx hours. 
     API Version: (prod)v.x.x.x                                                   
  • If you attempt to send a file that is too large, you will receive the following message.

List Files

In order to know that your followers have sent you files, you can list the current files available for download.
List Files.

  • Open a terminal/command line interface
  • Navigate to the folder containing the downloaded client app
  • Execute the following command to follow an identity:
     > oal list <identity>
  • If the list is successful, you will receive the following message.
     name:<name of file>,size:<total encrypted bytes> 
     . . .                                            
     Result: 200, Reason: Sent                        
     API Version: (prod)v.x.x.x                       
  • If the list is unsuccessful, you will receive the following message.
     Result: 403, Reason: Unauthorized 
     API Version: (prod)v.x.x.x        

Get Files

Getting files your followers have sent you is the second part of the file transfer.
Get Files.

  • Open a terminal/command line interface
  • Navigate to the folder containing the downloaded client app
  • Execute the following command to follow an identity:
     > oal get <identity> <filename> 
  • If the transfer is successful, you will receive the following message.

     <total decrypted bytes> successfully written to ./identity/<identity>/<filename> 
     Result: 200, Reason: OK                                                          
     API Version: (prod)v.x.x.x                                                       
  • Once the file is downloaded and decrypted, you can view the contents at the location specified in the GET message.
  • The identity specified in this command is YOUR identity, not the identity of the follower.

Delete Files

Delete files your followers have sent you.
Delete Files.

  • Open a terminal/command line interface
  • Navigate to the folder containing the downloaded client app
  • Execute the following command to follow an identity:
     > oal delete <identity> <filename> 
  • If the delete is successful, you will receive the following message.
     Result: 200, Reason: OK    
     API Version: (prod)v.x.x.x 
  • Once the file is deleted it is no longer available for download and will not show up in a LIST command results.
  • The identity specified in this command is YOUR identity, not the identity of the follower.

Display Info About Your Identities and Followers

Display Info About Your Identities and Followers.
Display Info.

  • Open a terminal/command line interface
  • Navigate to the folder containing the downloaded client app
  • Execute the following command to follow an identity:
     > oal info  
  • If the info is successful, you will receive the following message.
     Version: v.x.x.x           
     Build Type: production     
     Identities: . . .          
     Following:  . . .          
     Next available get: . . .  
     Next available put: . . .  
     Result: 200, Reason: OK    
     API Version: (prod)v.x.x.x 
  • Once the file is deleted it is no longer available for download and will not show up in a LIST command results.
  • The identity specified in this command is YOUR identity, not the identity of the follower.

Set Your Credit Card Payment Info

Set your credit card payment info to access commands more frequently.
Set Your Credit Card Payment Info.

  • Open a terminal/command line interface
  • Navigate to the folder containing the downloaded client app
  • Execute the following command to follow an identity:
     > oal set-payment <cc number> <exp date (mm/yyyy)> <CVC>  
  • If the set payment is successful, you will receive the following message.
     Result: 201, Reason: Payment Set 
     API Version: (prod)v.x.x.x       
  • The credit card info is checked by Stripe for validity.
  • If you attempt to set the credit card information after you have already added a credit card you will get the following message.
     Result: 409, Reason: Payment Already Set 
     API Version: (prod)v.x.x.x               
  • If you wish to change your credit card info, you can add "force" as the last argument in the command. This will force Stripe to override your current credit card with the new information. If the override is successful, you will receive the message above.

Paying For Transactions

After you set your payment (see above), you can pay for transactions. These are registering more than one identity, and getting and putting more frequently than the free schedule allows.
Paying For Transactions.

  • Open a terminal/command line interface
  • Navigate to the folder containing the downloaded client app
  • Execute any of the commands that require payment adding "auth-payment" as the last argument. Example:
     > oal get <identity> <filename> auth-payment 
  • If the charge to your credit card is successful, you will receive the appropriate success message for the command, and your credit card will be charged.
